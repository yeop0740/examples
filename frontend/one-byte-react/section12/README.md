### 페이지 라우팅
경로에 따라 알맞은 페이지를 렌더링 하는 과정

서버에서 각각의 페이지를 가지고 있고 요청 시 해당 페이지(html)을 반환하는 방식을 MPA(Multi Page Application) 이라고 한다.
MPA 는 서버 사이드 렌더링 이라고도 부른다. 정확히는 여러 페이지(html)를 가지고 있는 것을 MPA, 요청할 때 페이지(html)를 만들어서 제공하는 것을 SSR 이라고 한다.

react 는 MPA 방식을 따르지 않는다. 쾌적한 페이지 이동을 제공하기 어렵기 때문이다.

MPA 의 동작 방식
클라이언트에서 페이지 요청 -> 서버에서 적합한 파일 선택 및 반환 -> 클라이언트에서 이전 페이지 버리고 새로운 페이지를 렌더링한다.
위의 과정에서 페이지를 새로고침하는 것 처럼 깜빡인다.
MPA 는 응답으로 받은 페이지(html) 단위로 렌더링 하므로 동일한 구성 요소(변하지 않는 부분)또한 모두 렌더링 하게 된다.
동시 접속사 수에 의해 서버의 부하가 심해진다.

리액트는 SPA(Single Page Application)
페이지 이동이 매끄럽고 효율적임.
다수의 사용자가 접속해도 크게 상관 없음
페이지는 오직 하나, 리액트 컴포넌트, 기타 기능 들이 있다.

SPA 동작 방식
클라이언트에서 페이지 요청 -> 서버에서 index.html(react code 만 가동하는 페이지) 반환, 추가적으로 js 파일들 전달 -> 클라이언트에서 번들링된 파일을 직접 실행한다. 이때 컴포넌트가 렌더링 한다.
클라이언트 사이드 렌더링

동적 경로
동적인 데이터를 포함하고 있는 경로
`post/1`, `post/2`, `post/3` 등

- url parameter
  - path string 형태
  - 변경되지 않는 값을 주소로 명시하기 위해 사용됨
- query string
  - ? 뒤에 키 밸류 형태
  - 검색어 등의 자주 변경되는 값을 주소로 명시하기 위해 사용

**요구사항**
- "/" : 모든 일기를 조회하는 Home
- "/new" : 새로운 일기를 작성하는 페이지
- "/diary" : 일기를 상세히 조회하는 페이지
