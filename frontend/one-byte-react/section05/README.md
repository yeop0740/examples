원래 js/ts 에서는 태그를 리턴하는 것은 문법적으로 잘못 되었다고 인식한다.
하지만, jsx/tsx 문법적으로는 문제되지 않는다. js/ts 와 html 을 같이 사용할 수 있다.
jsx/tsx 를 사용하면 동적으로 특정 변수 값을 html 로 렌더링 할 수 있다. 즉, html 내부에 js/ts 변수를 사용할 수 있다.

jsx/tsx 주의사항
1. 중괄호 내부에는 js/tx 표현식만 사용할 수 있다(값으로 평가되는 코드).
2. 숫자, 문자열, 배열 값만 렌더링 된다. 불리언, undefined, null 값은 오류를 발생시키지 않지만, 렌더링 되지 않는다. 객체를 렌더링 시도할 때 오류가 발생한다.
3. 모든 태그는 닫혀있어야 한다.
4. 최상위 태그는 반드시 하나여야 한다(빈 태그를 사용할 수도 있음).

### props

컴포넌트를 재사용하여 사용하는 경우 컴포넌트를 구성하는 데이터를 변경하여 넣어줘야한다.
리액트에서 부모 컴포넌트는 자식 컴포넌트에게 값을 전달할 수 있다. 이때 컴포넌트에 전달된 값을 props 라고 부른다.

### 이벤트 핸들링
이벤트 발생 시 처리하는 것
ex) 사용자가 button 클릭

이벤트 핸들러에는 이벤트 객체가 전달된다. 이 이벤트 객체는 모든 브라우저에서 사용할 수 있도록 규격화된 형태이다.

### state
현재 가지고 있는 형태나 모양을 정의
변화할 수 있는 동적인 값
리액트의 컴포넌트는 state 를 가질 수 있다(0 ~ 여러개).
컴포넌트의 state 가 변경되면 react 가 이를 감지하여 리렌더링 한다.

일반 변수를 사용하지 않고 state 를 사용하는 이유 -> 해당 변수가 변경되더라도 리렌더링 되지 않는다.
리액트 컴포넌트는 부모로부터 받는 props 가 변경되면 리렌더링이 발생한다.

**리렌더링 케이스**
1. 자신이 관리하는 state 값이 변경되었을 때
2. 자신이 받는 props 값이 변경되었을 때
3. 부모 컴포넌트가 리렌더링 될 때
